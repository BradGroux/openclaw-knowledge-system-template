# Knowledge System Maturity Model

## L1 — Ad Hoc
- Notes exist but no standard structure
- Retrieval depends on individual memory

## L2 — Structured Raw Capture
- Raw snapshots stored by stream/date
- Metadata convention exists and is mostly followed

## L3 — Compiled with Lineage
- Synthesized docs per stream
- Compiled pages link to raw evidence
- Repeated asks answered from corpus first

## L4 — Governed + Automated
- Scheduled compile runs
- Contradiction/freshness quality gates
- Weekly synthesis and trend metrics

## L5 — Decision-Grade Memory
- High confidence in evidence lineage
- Quantified impact (faster retrieval, fewer repeats)
- Standardized review + publishing workflow
